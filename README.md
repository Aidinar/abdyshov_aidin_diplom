# Math Sense — семантический анализатор математических PDF-документов

Math Sense — это веб-приложение, которое превращает обычные математические PDF-документы в интерактивные интеллектуальные ресурсы.

Система автоматически:

- Извлекает текст из PDF (клиентская обработка, файл не покидает браузер)
- Выполняет семантическую аннотацию на основе онтологии OntoMathPRO
- Подсвечивает математические термины в тексте
- Рендерит формулы через KaTeX
- Строит семантические связи между документами (коэффициент Жаккара)
- Даёт рекомендации похожих документов
- Предоставляет интеллектуального помощника на базе локальной LLM (Ollama)

Разработано в рамках выпускной квалификационной работы (ВКР) по направлению «Интеллектуальные информационные аналитические системы».

## Основные возможности

### Функционал
- Клиентская обработка PDF — файл не покидает браузер пользователя (конфиденциальность)
- Семантическая разметка на основе онтологии OntoMathPRO (хранится в GraphDB)
- Двусторонняя навигация между текстом и списком терминов (клик → прокрутка)
- Рендеринг LaTeX-формул через KaTeX (строчные и выключные)
- Семантические связи документов по коэффициенту Жаккара
- Рекомендательная система похожих документов (топ-10)
- LLM-помощник (Ollama + fallback на определение из онтологии)
- История анализов и общая статистика
- Светлая/тёмная тема, адаптивный интерфейс

### Технологический стек

| Компонент       | Технологии                          |
|-----------------|-------------------------------------|
| Backend         | Node.js + Express                   |
| Реляционная БД  | PostgreSQL                          |
| Графовая БД     | GraphDB (онтология OntoMathPRO, SPARQL) |
| LLM             | Ollama (модель llama3.2:1b)         |
| Frontend        | Чистый JavaScript (SPA), pdf.js, KaTeX, Font Awesome |
| API             | RESTful                             |

## Запуск

### 1. Предварительные требования

Установите следующие компоненты:

- **Node.js** (версия 18 или выше)
- **PostgreSQL** (версия 14 или выше)
- **GraphDB** — Community Edition (бесплатная)
- **Ollama** (для локального запуска LLM)

### 2. Клонирование и установка зависимостей

```
git clone <URL_репозитория>
cd mathsense-backend
npm install
```

### 3. Настройка окружения

Создайте файл .env в корне backend со следующим содержимым:env

```
PORT=3001
DB_HOST=localhost
DB_PORT=5432
DB_NAME=mathsense
DB_USER=postgres
DB_PASSWORD=your_password_here   # замените на свой пароль
GRAPHDB_URL=http://localhost:7200/repositories/mathsense-ontology2
```

### 4. Подготовка баз данных

#### 4.1. PostgreSQL

Запустите PostgreSQL и создайте базу данных (скрипт сделает это автоматически, но можно и вручную)


```
sudo -u postgres psql
CREATE DATABASE mathsense;
\q
```

Затем выполните инициализацию таблиц
```
node setup-db.js
```

Скрипт создаст следующие таблицы:
 - files — метаданные загруженных PDF
 - terms — найденные термины с определениями и контекстом
 - file_connections — кэш попарных семантических связей


#### 4.2. GraphDB (онтология)

1. Запустите GraphDB.
2. Откройте веб-интерфейс: http://localhost:7200.
3. Создайте новый репозиторий:
    - Имя: mathsense-ontology2
    - Тип: GraphDB Repository

4. Загрузите онтологию:
    - Перейдите в Import → Upload files
    - Выберите файл OntoMathPRO2.ttl из корня проекта
    - Нажмите Upload

5. Проверьте доступность репозитория по адресу:

```
http://localhost:7200/repositories/mathsense-ontology2
```

### 5. Установка и запуск Ollama

Скачивание модели (около 0,8 ГБ)

```
ollama pull llama3.2:1b

```

Запуск сервиса (если не запущен)

```

ollama serve

```

Убедитесь, что Ollama отвечает

curl http://localhost:11434/api/tags

### 6. Запуск сервера

```
node server.js

```

Сервер запустится на http://localhost:3001.

### 7. Запуск клиентской части
Откройте файл index2.html в браузере.
Рекомендация: используйте Live Server в VS Code (или любой другой локальный сервер), чтобы избежать проблем с CORS при запросах к API.

### API Эндпоинты


| Метод | Эндпоинт                          | Описание                                      |
|-------|-----------------------------------|-----------------------------------------------|
| GET   | `/api/health`                     | Проверка работы сервера                       |
| GET   | `/api/ontology/terms`             | Получение всех терминов из онтологии (кэш 5 мин) |
| POST  | `/api/analyze/save`               | Сохранение результатов анализа (файл + термины) |
| GET   | `/api/stats`                      | Общая статистика (количество файлов, терминов) |
| GET   | `/api/files/history`              | История последних 50 файлов                   |
| GET   | `/api/files/all`                  | Список всех файлов                            |
| GET   | `/api/graph/global`               | Глобальный граф связей (с кэшем)              |
| GET   | `/api/graph/file/:id/connections` | Связи для конкретного файла                   |
| POST  | `/api/ai/recommend`               | Рекомендации похожих документов               |
| POST  | `/api/llm/ask`                    | Запрос к LLM-помощнику (Ollama)               |

### Примечания по использованию

- **Ручной ввод текста** — переключитесь на вкладку «Редактор текста», вставьте текст и нажмите «Анализировать текст».
- **LLM-помощник** — работает на вкладке редактора. Задавайте вопросы на русском языке (например, «Что такое абелева группа?»).
- **Рекомендации** — появляются автоматически после анализа и сохраняются в базе.
- **Тёмная тема** — переключается кнопкой в правом верхнем углу.

### Ограничения

- Сканированные PDF — не распознаются (нужен OCR).
- Многоколоночная вёрстка — может нарушаться порядок строк (ограничение pdf.js).
- Грамматические вариации — словарный метод не распознаёт падежи и числа (снижает полноту).
- Масштабируемость — вычисление связей имеет сложность O(N²), для >100 файлов требуется оптимизация.
